# CLAUDE.md — ansgarharmeier

Personal portfolio site for Ansgar Harmeier (HubSpot CRM / Automation / AI & Agents consultant, Düsseldorf).

- **Live:** https://ansgarharmeier.vercel.app · **Repo:** https://github.com/Aharmeier/ansgarharmeier · **Vercel scope:** `ansgarhs-projects`
- **Stack:** plain static site — no framework, no build step, **zero runtime dependencies** (everything is in `index.html`; only Google Fonts are fetched). It was originally generated by Claude's "omelette" design runtime, but the design-runtime scaffolding (image-slot web component + React/Babel "Tweaks" panel) has been stripped out.

## Files

| File | Purpose |
|------|---------|
| `index.html` | The whole site — markup, design tokens, CSS, EN/DE i18n strings, SEO metadata (canonical, OG/Twitter tags, JSON-LD `Person` schema), and behavior (theme/font/lang toggles). |
| `portrait-source.jpg` | **Gitignored.** Raw camera original (6000×4000, Canon EOS R6m2) — the master file for regenerating every tile below. Not deployed; keep it locally. |
| `portrait.jpg` | 1600×1066 export, derived from `portrait-source.jpg`. Used as the OG/Twitter share image and the `<img>` fallback for browsers without `<picture>` support. |
| `portrait-{480,800,1200,1600,2400}.{jpg,webp,avif}` | Responsive hero portrait tiers served via `<picture>` (AVIF → WebP → JPG fallback, matching `srcset`/`sizes`). File sizes range ~24KB (480w AVIF) to ~370KB (2400w AVIF) depending on which tile the browser picks — still well under the original single 272KB JPEG for the common case. |
| `sitemap.xml` | Single-URL sitemap, referenced from `robots.txt`. |
| `robots.txt` | Allows all crawlers; points to `sitemap.xml`. |
| `favicon.svg` | AH monogram favicon (brand blue `#1f3a8a`). |
| `apple-touch-icon.png` | 180×180 PNG home-screen icon (iOS ignores SVG apple-touch-icons). |

## Working notes

- **Run locally:** `python3 -m http.server 8765` → http://localhost:8765/ . Opening via `file://` works too now that there's no Babel/JSX fetch.
- **Deploy:** `vercel --prod` from this dir (project is linked; scope `ansgarhs-projects`). Deploys upload local files directly — not yet wired to auto-deploy from GitHub.
- Default look is hard-coded on `<html data-theme="light" data-font="grotesk">` with accent `--accent-base:#1f3a8a`. The old design "Tweaks" panel that switched these at runtime was removed; change them in the markup/CSS. The `html[data-font="serif"]`/`"plex"` CSS rules are dead code left over from that panel — their Google Fonts (Newsreader, Space Grotesk, IBM Plex Sans) were dropped from the `<link>` import since nothing sets `data-font` to those values anymore, so don't rely on them without re-adding the font import first.
- **Email buttons** (hero "Email me" + contact card) are plain `mailto:mail@ansgarharmeier.de` links with a URL-encoded prefilled `?subject=Anfrage über ansgarharmeier.de` — no JS, opens the OS mail client. The subject is static (not i18n'd). If a visitor reports "nothing happens" on click, it's almost always a missing default mail handler on their OS/browser, not a site bug — the link itself is a plain anchor with no JS interception.
- **Regenerating portrait variants** (if `portrait-source.jpg` changes): resize tiers with `sips -Z <width> -s formatOptions 92 portrait-source.jpg --out portrait-<width>.jpg`, then encode `cwebp -q 92` and `avifenc -q 82 -s 4` per tier (widths: 480/800/1200/1600/2400). Refresh `portrait.jpg` too via `sips -Z 1600 -s formatOptions 92 portrait-source.jpg --out portrait.jpg`.
- **The `sizes` attribute on the portrait `<picture>` is intentionally NOT `420px`/`100vw` matching the box's literal CSS width** — it's `(max-width: 820px) 130vw, 1000px`. The hero photo box is much taller than it is wide, and the source photo is landscape, so `object-fit: cover` zooms in far more than the box's width alone suggests (the crop is height-driven). A width-matching `sizes` value makes browsers under-fetch and upscale the image even though larger tiles exist — this cost a full round of "why is the photo still blurry" debugging (see commit `712e700`). Don't "simplify" this back to the box width.
- The custom domain `ansgarharmeier.de` is **not** connected to this Vercel project — DNS isn't manageable by the site owner, so it stays an external 301 redirect (Apache, third-party registrar) straight to `ansgarharmeier.vercel.app`. Canonical/OG/JSON-LD URLs in `index.html` deliberately point at the `.vercel.app` URL for that reason — that's the URL Google will actually index, since 301 redirects pass ranking signal to the target anyway. Don't "fix" these back to the `.de` domain unless the domain gets properly wired up in Vercel.
